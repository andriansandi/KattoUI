# AI Documentation Guide

## When to Update Docs

- New architecture or breaking change: update `ARCHITECTURE.md`, `rfcs/`, or `adr/`.
- New theme/provider/plugin SDK feature: update `THEMING.md`, `PROVIDER_API.md`, or `PLUGIN_SYSTEM.md`.
- User-facing feature: update `README.md` and relevant screenshots/placeholders.
- Security change: update `SECURITY.md` and `.ai/security.md`.

## Conventions

- Use clear, professional prose.
- Keep README quick-start short; move detail to dedicated docs.
- Include code examples where helpful.
- Keep ADRs concise: context, decision, consequences.
- Keep RFCs thorough: motivation, proposal, alternatives, adoption plan.
