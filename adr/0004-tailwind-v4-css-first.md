# ADR 0004: Use Tailwind CSS v4 with CSS-First Configuration

| Field | Value |
|-------|-------|
| Title | Use Tailwind CSS v4 with CSS-First Theming |
| Date | 2026-07-02 |
| Status | Accepted |

## Context

KattoUI's visual system must support runtime theme switching, installable themes, and a single token contract shared between the application and authentication UI. Tailwind CSS v4 introduces a CSS-first configuration model where theme values are defined in CSS using `@theme` and can reference CSS custom properties. This aligns naturally with a CSS-variable based theme engine.

## Decision

We will use **Tailwind CSS v4** with **CSS-first configuration**. Theme tokens will be expressed as CSS custom properties generated by `packages/design-system`, and Tailwind's `@theme` block will map those variables to utility classes.

## Consequences

### Positive

- Runtime theme switching only requires updating `data-theme` and the generated CSS variable layer; no Tailwind rebuild is required.
- Themes are plain data (token JSON) plus generated CSS, making plugin-supplied and user-installed themes straightforward.
- CSS variables are framework-agnostic, so plugin authors can use them without adopting a specific styling solution.
- Reduced JavaScript runtime for styling compared to CSS-in-JS libraries.

### Negative / Trade-offs

- Tailwind v4 is newer than v3; tooling, plugins, and community resources are still maturing.
- Custom tooling is required to generate CSS variable layers from theme JSON.
- Migration may be required if Tailwind v4 conventions change during beta/early releases.
